\subsubsection[{dsfmt\_\-fill\_\-array\_\-close1\_\-open2}]{\setlength{\rightskip}{0pt plus 5cm}void dsfmt\_\-fill\_\-array\_\-close1\_\-open2 ({\bf dsfmt\_\-t} $\ast$ {\em dsfmt}, \/  double {\em array}\mbox{[}$\,$\mbox{]}, \/  int {\em size})}}
\label{dSFMT_8c_abeef257039f3b95b7475bbecdb521b89}
This function generates double precision floating point pseudorandom numbers which distribute in the range \mbox{[}1, 2) to the specified array\mbox{[}\mbox{]} by one call. The number of pseudorandom numbers is specified by the argument {\bfseries size}, which must be at least (SFMT\_\-MEXP / 128) $\ast$ 2 and a multiple of two. The function \hyperlink{dSFMT_8h_a006127d9c8946a9e1ed89de92eb48e64}{get\_\-min\_\-array\_\-size()} returns this minimum size. The generation by this function is much faster than the following fill\_\-array\_\-xxx functions.

For initialization, \hyperlink{dSFMT_8h_a8dc496135101ef4140c7e790bf7474cd}{init\_\-gen\_\-rand()} or \hyperlink{dSFMT_8h_a46947b547a4f7d7758a682298f7ec0ce}{init\_\-by\_\-array()} must be called before the first call of this function. This function can not be used after calling genrand\_\-xxx functions, without initialization.


\begin{DoxyParams}{Parameters}
\item[{\em dsfmt}]dsfmt state vector. \item[{\em array}]an array where pseudorandom numbers are filled by this function. The pointer to the array must be \char`\"{}aligned\char`\"{} (namely, must be a multiple of 16) in the SIMD version, since it refers to the address of a 128-\/bit integer. In the standard C version, the pointer is arbitrary.\item[{\em size}]the number of 64-\/bit pseudorandom integers to be generated. size must be a multiple of 2, and greater than or equal to (SFMT\_\-MEXP / 128) $\ast$ 2.\end{DoxyParams}
\begin{DoxyNote}{Note}
{\bfseries memalign} or {\bfseries posix\_\-memalign} is available to get aligned memory. Mac OSX doesn't have these functions, but {\bfseries malloc} of OSX returns the pointer to the aligned memory block. 
\end{DoxyNote}
