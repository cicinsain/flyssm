Detailed Description

Three function implementations that read, write and remove the state file for an annealing run.

Copyright (C) 1989-2003 John Reinitz, 2009-2013 Damjan Cicin-Sain, Anton Crombach and Yogi Jaeger

The frequency with which state are saved can be chosen by the command line op- tion -b (for backup stepsize). The state file is very useful for the case when long annealing runs have to be interrupted or crash for some reason or another. The run can then be re- sumed by indicating the state file as an additional argument to fly_sa.
