00073 /* These are the Lam parameters: *******************************************
00074  * (see King-Wai Chu's thesis for a detailed explanation)                  *
00075  *                                                                         *
00076  * lambda:              overall annealing accuracy                         *
00077  * lambda_mem_length_u: weighting factor for estimating mean energy        *
00078  * lambda_mem_length_v: weighting factor for estimating standard deviation *
00079  * initial_moves:       number of initial moves to randomize initial state *
00080  *                      of the system and gather initial statistics        *
00081  * tau:                 number of moves between updating statistics and    *
00082  *                      Lam parameters                                     *
00083  * freeze_count:        number of times system needs to be 'Frozen()' be-  *
00084  *                      fore we stop annealing                             *
00085  * update_S_skip:       number of iterations between changing the inverse  *
00086  *                      temperature; exists for increasing code efficiency *
00087  *                      and is probably obsolete now                       *
00088  * control:             OBSOLETE, used to be the control parameter for the *
00089  *                      move generation; acceptance statistics now live in *
00090  *                      move(s).c, since they are problem-specific         *
00091  * criterion:           defines the limits of what counts as 'Frozen()';   *
00092  *                      depends on the stop_flag (see below), i.e. it is   *
00093  *                      either a fixed energy (for discrete problems), a   *
00094  *                      limit to the change in absolute energy or the lim- *
00095  *                      it of proportional change in energy (good for con- *
00096  *                      tinous search spaces)                              *
00097  * mix_interval:        the mixing interval for parallel code in 'tau'     *
00098  *                      units, i.e. 100 means mix every 100 tau            *
00099  *                                                                         *
00100  ***************************************************************************
00101  *                                                                         *
00102  * NOTE: the stopping criterion is actually defined by the combination of  *
00103  *       'freeze_count' and 'criterion' below plus the stop_flag which     *
00104  *       defines the type of the criterion (absolute or proportional);     *
00105  *       usually when we talk about criterion (e.g. Chu, 2001) we mean     *
00106  *       only the value of criterion (kappa) though, since 'freeze_count'  *
00107  *       and the stop_flag should always be constant for specific problems *
00108  *                                                                         *
00109  ***************************************************************************/
